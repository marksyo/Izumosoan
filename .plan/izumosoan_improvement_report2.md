改善が進む一方で、法人顧客が問い合わせ前に行う「実在性・信頼性の確認」において、依然として不足している要素がある。以下に優先度順で詳述する。

## 2. 改善提案

### ■ 優先度：高（今すぐ対応）

---

#### 課題 A：会社の基本法人情報が存在しない

**現状**

プライバシーポリシーページで社名が `出雲創安合同会社` と確認できるが、これ以外の法人情報がサイト全体のどこにも存在しない。会社概要ページは理念・ビジョン・哲学のみで構成されており、法人としての基本情報が一切ない。

BtoB取引における法人顧客は、問い合わせ前に必ず以下を確認する。

- 正式社名（法人格）
- 代表者名
- 設立年月日
- 資本金
- 法人番号（国税庁法人番号サイトで照合できるか）

これらが見つからないと、「怪しい会社かもしれない」という印象を与え、問い合わせ自体を断念されるリスクがある。

**改善案**

会社概要ページに以下のような法人情報テーブルを追加する。

```
| 会社名     | 出雲創安合同会社                         |
| 設立       | ○○年○月                                |
| 代表者     | ○○ ○○                                 |
| 資本金     | ○○万円                                 |
| 従業員数   | ○名                                    |
| 所在地     | 東京都葛飾区東立石4-37-8 [Googleマップ] |
| 法人番号   | ○○○○○○○○○○○○○                        |
| 電話       | 050-1258-9122                          |
| メール     | info@izumosoan.com                     |
```

---

#### 課題 B：代表者・チームの顔が一切見えない

**現状**

会社概要ページには「創業チームは長年にわたり現場工学に携わってきたエンジニアで構成されています」とあるが、何名なのか、誰なのか、どんな経歴なのかが全くわからない。「20年以上の現場経験」も、誰の経験なのかが不明。

BtoBサービスの場合、法人顧客は発注先の「担当者の顔と実力」を確認したい。特に創業期の会社においては、チームの信頼性が受注の可否を大きく左右する。

**改善案**

会社概要ページに「チームメンバー」セクションを追加する。

```
【記載内容（例）】
・代表者または主要エンジニアの顔写真
・氏名（または「代表エンジニア」などの役職でも可）
・略歴：前職・専門領域・経験年数
・保有資格（例：電気工事士・CCNA・PMP等）
・一言メッセージ（任意）
```

顔写真の掲載が難しい場合でも、「現場エンジニア○名体制」「平均経験年数○年」などの情報を明記するだけで印象は大きく変わる。

---

#### 課題 C：ナビゲーション構成がページによって異なる

**現状**

各ページのナビゲーションを確認すると、メニュー項目の順序と構成が統一されていない。

| ページ | ナビゲーション順序 |
|--------|------------------|
| トップページ | ホーム / サービス / 製品 / 実績 / 会社概要 / お問い合わせ |
| 会社概要 | ホーム / 製品 / サービス / 会社概要 / お問い合わせ（実績なし） |
| サービス | ホーム / 製品 / サービス / 会社概要 / お問い合わせ（実績なし） |
| 実績 | ホーム / 会社概要 / サービス / 実績 / お問い合わせ（製品なし） |
| プライバシーポリシー | ホーム / サービス / 製品 / 実績 / 会社概要 / お問い合わせ |

「実績」メニューが一部のページにしかない、「製品」の位置が統一されていない、など複数の不整合がある。

**改善案**

全ページで以下のナビゲーションに統一する。

```
ホーム ／ サービス ／ 製品 ／ 実績 ／ 会社概要 ／ お問い合わせ
```

また、フッターメニューについても、プライバシーポリシーページのフッターには「製品情報」リンクがなく、他ページのフッターとも構成が異なっているため、統一が必要。

---

#### 課題 D：会社概要ページのog:site_nameに旧表記が残っている

**現状**

会社概要ページのメタ情報を確認すると、`meta-og:site_name` が `出云创安 Izumo Soan`（簡体字）のままになっている。トップページや実績ページは `出雲創安 Izumo Soan`（正しい日本語表記）になっており、統一されていない。

```
【問題あり】会社概要ページ：出云创安 Izumo Soan
【正しい表記】トップページ ：出雲創安 Izumo Soan
```

サービスページのog:site_nameも同様に `出云创安 Izumo Soan` になっており、要修正。

**改善案**

全ページの `og:site_name` を `出雲創安 Izumo Soan` に統一する。

### ■ 優先度：中（1ヶ月以内に対応）

#### 課題 E：実績ページの「お客様の声」に属性情報が不足している

**現状**

実績ページ下部に3件のお客様の声が掲載されているが、いずれも「製造業 設備部長」「ITサービス 運用マネージャー」「建設会社 現場監理」という属性のみで、上部に掲載されているプロジェクト事例との紐付けがない。

プロジェクト事例の中には引用コメント付きのものもあるが、下部の「お客様の声」セクションとの関係が不明確なため、読者が混乱する。

**改善案**

- 「お客様の声」の各コメントを、対応するプロジェクト事例と明確に紐付ける（例：「AGV導入プロジェクト担当者より」という見出しを追加）
- または、プロジェクト事例内のコメントと重複しているなら、下部セクションを整理・統合する

---

#### 課題 F：お問い合わせフォームに担当者情報がない

**現状**

フォームの横に「1営業日以内にご返信」の案内が追加されたが、誰が対応するのかが不明。匿名の問い合わせ先よりも、担当者の名前や写真があるほうが心理的安心感が高まる。

**改善案**

フォーム付近に以下を追加する。

```
担当：エンジニアリング営業 ○○（または「現場技術チーム」）
お急ぎの場合：050-1258-9122
```

担当者の顔写真（課題Bで追加するもの）を転用できれば効果的。

---

#### 課題 G：サービスページにCTAがない

**現状**

サービスページは12項目のサービスを詳細に説明しているが、ページ内にお問い合わせへの導線（CTAボタン）が一切ない。ユーザーが「このサービスを依頼したい」と思った瞬間にアクションできない。

**改善案**

サービスページの末尾（または各サービスカードの下）に以下のCTAを追加する。

```
このサービスについて相談する → [お問い合わせ]
```

また、実績ページの「あなたの現場の課題を解決します」セクションのようなCTAブロックをサービスページにも追加する。

---

#### 課題 H：製品ページへの誘導が弱い

**現状**

「IZUMO SafeOps」と「IZUMO SiteBridge」という自社製品があるが、トップページのメインコンテンツ内にこれらへの案内がほぼない（フッターのみ）。

会社のオリジナル製品は差別化の強力な武器であり、前面に出すべきコンテンツ。

**改善案**

トップページの「私たちのソリューション」セクション、またはその直下に製品紹介ブロックを追加する。

```
【自社製品】
IZUMO SafeOps - 現場安全運営調整プラットフォーム [詳細を見る]
IZUMO SiteBridge - スマート現場の運用改善プラットフォーム [詳細を見る]
```

### ■ 優先度：低（3ヶ月以内に対応）

---

#### 課題 I：英語版・中文版のコンテンツ品質確認

**現状**

3言語対応（日本語・英語・中文）を強みとして掲げているが、各言語版のコンテンツ内容・品質が日本語版と同等かどうかが未確認。特に英語版は海外メーカー向け営業において重要。

**改善案**

- 英語版のすべてのページをネイティブチェックにかける
- 日本語版と同等の情報量・更新頻度を維持する
- 海外メーカー向けパートナーセクションは英語版で最も充実させる

---

#### 課題 J：技術コラム・ブログの開始

**現状**

専門性をアピールするコンテンツとして、技術記事が存在しない。創業期のBtoBサイトにおいて、コラム記事は「専門家としての信頼構築」と「SEO」の両方に効果がある。

**改善案**

月1〜2本のペースで以下のような記事を掲載する。

```
例：
・「OT/ITネットワーク統合でよくある5つの失敗例」
・「AGV導入前に確認すべき無線環境チェックリスト」
・「日本の工場に海外製PLC/ロボットを導入する際の注意点」
・「データセンターのネットワーク配線品質を決める3つのポイント」
```

## 4. まとめ

前回からの改善で、プライバシーポリシーの整備・住所の日本語統一・フォームへの返信案内追加・技術領域の免責表記など、複数の重要課題がすでに対応されている。これは大きな前進。

現時点での最大の課題は **「会社の法人情報と代表者・チームの顔が見えないこと」** である。法人番号・代表者名・設立年などは、受注前に顧客が必ず確認する情報であり、これが見当たらないと問い合わせ自体をやめてしまうリスクがある。

実績ページ（case-studies.html）はすでに課題・対応・成果・お客様の声が揃っており、クオリティが高い。この実績の質を「会社情報の透明性」と組み合わせることで、初めて訪問した法人顧客が安心して問い合わせできるサイトになる。
