# 出雲創安 Webサイト 信頼性向上 改善提案レポート

**対象URL：** https://izumosoan.com/new-site/ja/index.html  
**調査ページ：** トップページ / サービス / 実績 / 会社概要  
**作成日：** 2026年6月1日  

---

## エグゼクティブサマリー

出雲創安のサイトは、サービス内容・業種対応・技術領域の情報量は充実している。ただし、法人顧客が「問い合わせ前に行う実在性・信頼性の確認」に必要な情報が不足しており、現状では問い合わせを躊躇させる要因が複数ある。

特に創業期のBtoBサイトにおいて、顧客が最初に行うのは「この会社は本当に存在するのか」「頼んで大丈夫か」の確認であり、この検証をサイト上で完結させることが最優先課題である。

---

## 現状の強み（維持・強化すべき点）

- サービス内容が業種別・カテゴリ別に整理されており、わかりやすい
- 実績ページ（case-studies.html）は課題・対応・成果の構成が明快で説得力がある
- 「海外メーカー向けパートナー」という独自のポジショニングが明確
- 3言語対応（日本語・英語・中文）で多様な顧客層に対応
- 「ご相談の流れ」の5ステップが掲載されており、初回問い合わせのイメージが持てる

## 改善提案

### 優先度：高（今すぐ対応）

---

#### 1. 会社情報の不整合と欠如

**現状の問題**

フッターの住所表記が日本語と中国語の混在になっている。

```
現状（問題あり）：「日本东京都葛饰区东立石4-37-8」
修正後：「東京都葛飾区東立石4-37-8」
```

また、以下の基本法人情報がサイト上のどこにも存在しない。

- 正式会社名（法人格）
- 設立年月日
- 代表者名
- 資本金
- 従業員数
- 法人番号

**改善案**

会社概要ページに下記のような法人情報テーブルを追加する。

| 項目 | 内容 |
|------|------|
| 会社名 | 株式会社 出雲創安（または合同会社等、正式名称） |
| 代表者 | ○○ ○○ |
| 設立 | ○○年○月 |
| 資本金 | ○○万円 |
| 従業員数 | ○名 |
| 所在地 | 東京都葛飾区東立石4-37-8 [Googleマップ](#) |
| 法人番号 | ○○○○○○○○○○○○○ |
| 電話 | 050-1258-9122 |
| メール | info@izumosoan.com |

#### 2. プライバシーポリシーの不在

**現状の問題**

問い合わせフォームでメールアドレス・会社名などの個人情報を収集しているにもかかわらず、プライバシーポリシーページへのリンクが存在しない。これは個人情報保護法上の義務を果たしていない状態であり、法的リスクと信頼性の両面で問題がある。

**改善案**

- プライバシーポリシーページを新規作成し、フッターおよびお問い合わせフォームの直下にリンクを追加する
- フォームの送信ボタン付近に「プライバシーポリシーに同意の上、送信します」のチェックボックスを追加する

---

#### 3. チーム・人物情報の欠如

**現状の問題**

「20年以上の現場経験」と記載されているが、誰のことなのか、何名のチームなのかが全くわからない。BtoB取引では担当者の顔が見えないと信頼が生まれにくい。創業期のスタートアップほど、「どんな人が運営しているか」が重要な判断材料になる。

**改善案**

会社概要ページに「チーム・メンバー」セクションを追加し、以下を掲載する。

- 代表者または主要エンジニアの顔写真
- 氏名（または役職のみでも可）
- 略歴（前職・担当領域・経験年数）
- 保有資格（例：電気工事士、ネットワーク技術者、CCNA等）

顔写真が難しい場合でも、チームの人数や構成（例：「現場エンジニア5名、プロジェクトマネージャー2名」）を明記するだけで印象が変わる。

### 優先度：中（1ヶ月以内に対応）

---

#### 4. 実績カードに顧客の声が未連携（トップページ）

**現状の問題**

トップページの実績カード3件（データセンター、AGV、ERP）は規模・工期の記載はあるが、実績詳細ページの充実した情報（課題・成果・お客様の声）につながっていない。トップページだけ見ると「関東 完了」という情報しか得られず、説得力が弱い。

**改善案**

- 各実績カードに「詳細を見る」ボタンを目立たせ、実績詳細ページ（case-studies.html）への誘導を強化する
- またはトップページのカードに「成果の一言」を追加する

```
例：
物流倉庫AGV導入支援
台数：20台 ／ 工期：6ヶ月 ／ 効率：60%向上
「システムだけでなく、現場スタッフへの説明まで一体で進められたことが大きかった」（物流倉庫 運用責任者）
```

#### 5. ナビゲーションの不統一

**現状の問題**

ページによってナビゲーションのメニュー構成が異なっている。

| ページ | ナビゲーション内容 |
|--------|----------------|
| トップページ | ホーム / 会社概要 / サービス / 実績 / お問い合わせ |
| サービスページ | ホーム / **製品** / サービス / 会社概要 / お問い合わせ |
| 実績ページ | ホーム / 会社概要 / サービス / 実績 / お問い合わせ |

「製品」メニューがサービスページにだけ存在し、他ページには表示されない。また「会社概要」と「サービス」の順序がページによって異なる。

**改善案**

全ページで共通のナビゲーションに統一する。製品ページ（IZUMO SafeOps / SiteBridge）が存在するなら、全ページのナビに追加する。

```
推奨統一メニュー：
ホーム ／ サービス ／ 製品 ／ 実績 ／ 会社概要 ／ お問い合わせ
```

---

#### 6. 問い合わせフォームの信頼要素の不足

**現状の問題**

フォームが項目と送信ボタンだけで、以下の信頼要素が一切ない。

- 返信時間の目安
- 担当者名または担当部署
- 「押すと何が起きるか」の説明

**改善案**

フォーム付近に以下を追加する。

```
【返信について】
通常、1営業日以内にメールにてご返信いたします。
お急ぎの場合はお電話（050-1258-9122）でも受け付けています。
担当：エンジニアリング営業チーム
```

また「無料相談を予約」ボタンがトップページのヒーローにあるが、実際にはフォーム送信であり「予約」ではない。「まずはご相談ください」など実態に即した表現に変更することで、期待値のズレを防ぐ。

---

#### 7. 資格・パートナー認定の可視化

**現状の問題**

対応技術領域にCisco / Juniper / Siemens / Mitsubishiなどが記載されているが、正規パートナーなのか、単に作業経験があるのかが不明。顧客は「パートナー認定企業なら安心」と判断するため、未記載は機会損失になる。

**改善案**

- 取得済みの認定がある場合：メーカーのパートナーバッジ画像をロゴグリッドで掲載
- 認定がない場合：「対応実績あり（認定パートナーではありません）」と誠実に明記

誠実な表記は、認定がないことよりも信頼性を高める。

### 優先度：低（3ヶ月以内に対応）

---

#### 8. 技術コンテンツ（ブログ・コラム）の追加

**改善案**

以下のような技術記事を定期掲載することで、専門性の証明とSEO効果の両方が得られる。創業期のサイトで最も費用対効果の高い施策の一つ。

- 「OT/ITネットワーク統合でよくある5つの失敗例」
- 「AGV導入前に確認すべき無線環境チェックリスト」
- 「日本の現場に海外技術を導入する際のポイント」

---

#### 9. 英語・中文コンテンツの品質均一化

**現状の問題**

3言語対応を強みとして訴求しているが、各言語版のコンテンツ品質・情報量に差がある場合、海外メーカーからの問い合わせ時に信頼性を損なう。

**改善案**

- 英語版はネイティブチェックを入れ、自然なビジネス英語に整備
- 海外メーカー向けページ（"We bridge technology and Japanese worksites"）は英語版を特に充実させる
- 言語切替後に該当言語のページへ正しくリダイレクトされることを確認（現在、canonical URLが `/ja/` 以下に固定されている点を要確認）

---

#### 10. 製品ページの信頼性強化

**現状の問題**

フッターに「IZUMO SafeOps」「IZUMO SiteBridge」という自社製品へのリンクがあるが、これらの製品ページはトップページのナビには表示されておらず、発見しづらい。

**改善案**

- 製品ページをナビゲーションに追加
- 各製品ページに「この製品を実際に使った事例」へのリンクを追加し、実績との連携を強化する

## まとめ

出雲創安のサイトには「何ができるか」の情報は十分にある。次のステップは「誰がやるのか」「本当に実在するのか」「頼んで大丈夫か」という信頼の証拠を加えることである。

特に**プライバシーポリシーの欠如**と**住所の言語混在**は、今すぐ対応すべき問題であり、この2点だけでも修正することで第一印象が大きく改善される。

実績ページ（case-studies.html）はすでに課題・対応・成果・お客様の声が揃っており、完成度が高い。この質をトップページにも反映させることが、次の重要な施策となる。
